# Gameboy Color エミュレーター

BubiBoyLiteを正式名称とするが、実行ファイル名は **bbl**とする

## 特徴

* CLIのみ。引数で一時的な設定変更も可能
* Claude Codeなどに見られるTUIも提供する
* macOS, Linuxに対応。x86_64/amd64, arm64をカバー。
  （2026-07-16、ユーザー承認によりWindows/Raspberry Pi OS/FreeBSDは対応対象から外した。
  経緯はdocs/dev/phases/phase-10-cicd.md参照）
* キーボードショートカットで操作
* 高速動作（処理負荷が軽い）、バイナリサイズが小さい
* ゲームコントローラーでも操作できる

## 対応OS、アーキテクチャ

* macOS 13.5以上 (Xcode 26でビルドすることを想定) : x86_64, Appleシリコン
* Linux Ubuntu 22.04以上 : amd64, arm64

macOS版はユニバーサルではなく、別々のバイナリとする

## 技術スタック

* 言語はOdinを使用する。miseでインストール済み。
* 描画、音声、入力はSDL2を使用
* 静的リンクして実行ファイルのみで実行可能にする

## コマンドラインオプション

```shell
bbl [options] game.gbc
````

-h, --help　: コマンドラインの使い方を表示
-v, --version : バージョンを表示

--scale : 表示倍率。1,2,3,4,5,6,7,8倍まででそれ以上の数字は8に丸める。デフォルトは4。
--fullscreen : フルスクリーン。--scaleがあっても無視。画面に収まる最大の整数倍率で表示。
--shader : シェーダー。nearest, smooth。デフォルトはnearest。
--recent : 最近使ったファイルを表示して選択できる。ROMファイルの指定がある場合は、そちらを優先。

その他、開発時に役立つものがあれば提案してください。
TUIを活かすアイデアがあると助かります。


## 設定ファイルの場所

* 実行ファイルと同じ場所とする
* 起動時に設定ファイルがなければ、すべてデフォルト値で作成する

## セーブ、ステートファイルの場所

* セーブファイルは実行しているゲームROMファイルと同じ場所をデフォルトとする
* 場所は設定ファイルで変更可能

## CI/CD

* GitHubのworkflowによって行う。M88Mを参考にすること。~/dev/_Emu/M88M
* 開発時は、簡単にビルドができるよう、開発環境にあったスクリプトを用意する（sh）
* CI/CDでも同じスクリプトを使用することで差異のないようにしたい
* 頒布する形式は全てのプラットフォームでzipファイルとし、この中に必要ならライセンスファイルも同梱する
